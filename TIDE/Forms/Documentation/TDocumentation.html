Overview

T is my own language that is compiled to assembler code for the 8051 microcontroller.
You can download a simulator for the 8051 from the website from Mr. Holzer.
The actual compiler/IDE that is described in this documentation can be found here.
Doing anything not described in this documentation may result in unexpected behavior (or hopefully show you an error message).

TIDE

The TIDE is an IDE for T. For the language T see T.
The TIDE is divided in two tabs: The editor and the assembler tab.

The editor tab

Her you can type the T code. Syntax coloring is automatically activated.
If you want to see some IntelliSense press STRG+Space.

IntelliSense

IntelliSense automaitcally shows you relevant words you could insert.
It completes your current word, it doesn't necassarily type in the whole word.
It shows you keywords, variables and methods to use.
You can enter the currently selected word (normally the first one) with Enter/Tab or by double clicking it.
If you want to select another item you can scroll down with the keys or your mousewheel and select the new one by clicking it.

The toolbar

Here you can do some actions.

  • The first item is for compiling the document. If necassary it's saved and then compiled. After compiling the assembler tab with the compiled code is opened.
  • The second item is for running the document. It's firstly compiled, then the compiled code is copied and the simulator from Mr Holzer is opened.
  • The third item is for saving the document. If necassary it prompts you to select the path to save it.
  • The fourth item is for saving the document as well, but it always prompts you.
  • The fifth item is for opening a document. If necassary it prompts you to save the old one. You will then have to select the file to open.
  • The sixth item is for creating a new document. If necassary it prompts you to save the old one. Before compiling you'll then have to save the document.
  • The seventh item shows you this document.
  • The eighth item colors the whole document, what is normally automatically done.

Shortcuts

There exist, additionally to the standard shortcuts like STRG+C, some special shortcuts:

  • F5 - compiles & runs the document.
  • STRG+F5 - compiles the document
  • STRG+S - saves the document.
  • STRG+SHIFT+S - saves the document under a specific path.
  • STRG+O - opens another document.
  • STRG+N - creates a new document.
  • STRG+MOUSEWHEEL - changes the text size.

The assembler tab

Shows the compiled assembler code.
It automatically colors the assembler code.
You can't edit the code here, but you can copy and paste it into the simulator to execute.

T

T is the language the compiler is for. Here you will get a quick overview about it.

General

Every command takes one line of code:
int a
The whole code is case-insensitive.
int a
A := 5
Comments are done with ; and comment the line from its position on.

Declaration

A declaration uses the type and the name of the variable:
bool b
The value can directly get assigned with a normal assignment:
int i := 1

Assignment

An assignments writes a variable into the variable.
The variable name and the new value/other variable must be defined
int i
i := 1
int u
u := i
There exist some special assignments as well.

AddAssignment

The AddAssignment adds the variable on the right to the variable on the left.
Both must be of a Byte type.
int i := 1
i += 1

SubtractAssignment

The SubtractAssignment subtracts the variable on the right from the variable on the left.
Both must be of a Byte type.
int i := 1
i -= 1

MultiplyAssignment

The MultiplyAssignment multiplys the variable on the right to the variable on the left.
Both must be of a Byte type.
int i := 1
i *= 2

DivideAssignment

The DivideAssignment divides the variable on the left by the variable on the right.
Both must be of a Byte type.
int i := 2
i /= 2

ModuloAssignment

The ModuloAssignment makes the left variable modulo the right one and writes the result into the left one.
Both must be of a Byte type.
int i := 1
i %= 2

AndAssignment

The AndAssignment evaluates the logical result of the left variable and the right variable and writes it into the left variable.
Both must be of the same type.
bool b := true b S= false

OrAssignment

The OrAssignment evaluates the logical result of the left variable or the right variable and writes it into the left variable
Both must be of the same type.
bool b := false b |= true

Types

In general there are two types: Bit and Byte types.
As Bit type only the boolean exists, for Byte types integers, cintegers and characters.

Boolean

A boolean value is declared with bool.
bool b
It can contain the values true and false, normally it has the value false.
bool b
b := true
b := false
 You can access one bit of a byte with a dot.
int i
bool b := i.i

Integer

An integer can contain 8 bit positive integral numbers.
It is declared with int.
int i
i := 1

Cinteger

A cinteger can contain additionally to positive integers negative ones as well. Therefor it has a range from -128 ~ 127.
It is declared with cint.
cint ci
ci := -1
ci := 1

Character

A character can contain most of the ASCII characters.
It is declared with char.
char c
c := 'h'

Blocks

A block is an area in which specific variables are visible and where you can jump out with break.
block
  int i := 1
  break
  i++
endblock
or
{
  break
}

If block

The if block is an area in which the content is only then executed when the condition is true.
bool condition := 4 = 5
if [condition]
  condition := false
endif

While block

The while block is an area in which the content is repeated while the condition is true.
while [true]
  if [true false]
    break
  endif
endwhile

ForTil block

The ForTil block counts from the limit till zero and repeats it content during this.
fortil 10 i
  Port0 := i
endfortil

Method

A method is a block that can get called, can have a return type and can have parameters.
The jumping out is done with return.
int i := m [true]
 
method m [bool returning]
  if [returning]
    return 1
  endif
endmethod

Interrupt Service Routine

The Interrupt Service Routine (ISR) is a special method that is called on one of the six interrupts.
Actually normal code execution is normally executed while the microcontroller is waiting for an interrupt.
There exist two external interrupts (0/1) that listen on the ports 3.2/3.
isrexternal0
  return
endisr
Additionally there are two Timer and two Counter interrupts.
Actually you can only use them as one, so you can't use the timer interrupt 0 and the counter interrupt 0.
Both of them need a value to which they count. So in the given examples the timer counts ten times and then executes the stuff inside.
isrtimer0 10
  return
endisr
Or
isrcounter0 10
  return
endisr <

Operations

In most cases operations evaluate the result but don't automatically write the result into another variable!

Decrement

The decrement is one of the operations that directly changes the value of the variable.
It still returns the final value though.
Both parameters have to be of a Byte type.
int i
int a := i--

Increment

The increment is the other operation that directly changes the value of the variable.
It still returns the final value as well.
Both parameters have to be of a Byte type.
int i
int a := i++

Logical Not

The logical not returns the complement of the given variable/value.
bool b := !true
if [!b]
  Port0 := !1
endif

Equal comparison

The equal comparison evaluates wether both values/variables have the same value.
Both parameters have to be of a Byte type.
int i := 1
bool b := i = 2

UnEqual comparison

The unEqual comparison evaluates wether the values/variables don't have the same value.
Both parameters have to be of a Byte type.
int i := 1
bool b := i != 2

Bigger comparison

The bigger comparison evaluates wether the first value/variable is bigger than the second one.
Both parameters have to be of a Byte type.
int i := 1
bool b := 2 > i

Smaller comparison

The smaller comparison evaluates wether the first value/variable is smaller than the second one.
Both parameters have to be of a Byte type.
int i := 1
bool b := 2 < i

Logical and

The logical and evaluates the logical result of the first parameter and the second one.
int a := 1
int b := a S 11

Logical or

The logical or evaluates the logical result of the first parameter or the second one.
bool a := true
bool b := a | false

Sum

The sum operator evaluates the sum of the two parameters.
Both parameters have to be of a Byte type.
int a := 1
int b := a + 2

Subtract

The subtract operator evaluates the result of the second paramter subtracted from the first one.
Both parameters have to be of a Byte type.
int a := 2
int b := a - 1

Multiply

The multiply operator multiplies the two parameters.
Both parameters have to be of a Byte type.
int a := 2
int b := a * 3

Divide

The divide operator divides the first parameter with the second one.
Both parameters have to be of a Byte type.
int a := 4
int b := 4 / 2

Modulo

The modulo operator evaluates the result of the first parameter modulo the second one.
Both parameters have to be of a Byte type.
int a := 6
int b := a % 4

ShiftLeft

The shiftLeft operator evaluates the result of the first parameter shifted to the left by the second paramter.
The bits that get pushed out pop into the beginning again.
Both parameters have to be of a Byte type.
int a := 1
int b := a << 2

ShiftRight

The shiftRight operator evaluates the result of the first parameter shifted to the right by the second paramter.
The bits that get pushed out pop into the end again.
Both parameters have to be of a Byte type.
int a := 4
int b := a >> 2
